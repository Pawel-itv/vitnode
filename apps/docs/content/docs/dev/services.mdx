Services helps to separate the business logic from the GraphQL resolvers. It's a good practice to keep the resolvers as thin as possible.

import { Step, Steps } from 'fumadocs-ui/components/steps';

<Steps>
<Step>
## Create a Service

Let's create a simple logic for the GraphQL query or mutation.

```ts title="backend/plugins/blog/categories/show/show.service.ts"
import { Injectable } from '@nestjs/common';

@Injectable()
export class ShowBlogCategoriesService {
  show(): string {
    return 'Hello World!';
  }
}
```

</Step>
<Step>
## Provide into the Module

```ts title="backend/plugins/blog/categories/categories.module.ts"
import { Module } from '@nestjs/common';

import { ShowBlogCategoriesService } from './show/show.service';

@Module({
  providers: [ShowBlogCategoriesService], // [!code highlight]
})
export class BlogCategoriesModule {}
```

</Step>
</Steps>
